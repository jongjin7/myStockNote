## Executive Summary

### 제품 개요
**StockNote**는 개인 투자자가 여러 증권사에 분산된 주식 자산을 통합 관리하고, 투자 판단과 회고를 체계적으로 기록할 수 있는 개인용 웹 애플리케이션입니다.

### 핵심 가치 제안
> "투자 수익률이 아닌, 투자 판단의 기록과 복기를 통해 더 나은 의사결정을 돕는다"

### 주요 특징
- 📊 **통합 자산 관리**: 복수 증권사 계좌의 예수금과 보유 종목을 한 곳에서 관리
- 📝 **투자 노트**: 매수/매도 판단과 리서치 내용을 구조화하여 기록
- 🔄 **투자 복기**: 과거 투자 판단을 다시 읽으며 의사결정 개선
- ☁️ **클라우드 동기화**: Supabase 기반 실시간 데이터 동기화
- � **다중 기기 지원**: PC, 모바일, 태블릿 어디서나 접근 가능
- 🔒 **안전한 데이터**: 클라우드 자동 백업으로 데이터 손실 걱정 없음

### 타겟 사용자
- **주 사용자**: 복수 증권사 계좌를 사용하는 개인 투자자
- **투자 경험**: 초보~중급 (주린이)
- **투자 성향**: 국내/해외 주식, ETF, 배당주 중심
- **니즈**: 투자 판단 기록, 자산 통합 관리, 투자 복기

### MVP 목표
- **기간**: ASAP
- **핵심 지표**: 본인이 주 2회 이상 사용
- **성공 기준**: 투자 판단을 앱에서 찾아볼 수 있음

### 차별점
| 기존 도구 | StockNote |
|----------|-----------|
| HTS/MTS: 실시간 시세 중심 | ✅ 투자 판단 기록 중심 |
| 엑셀: 자유롭지만 구조화 어려움 | ✅ 구조화된 노트 시스템 |
| 메모앱: 범용 도구 | ✅ 주식 투자 특화 |
| LocalStorage: 기기별 분리 | ✅ 클라우드 동기화 (다중 기기) |

## 1. 문제 정의 (Problem Statement)

### 사용자 페인 포인트

#### 1.1 자산 분산 문제
- 증권사마다 앱/HTS가 분리되어 전체 자산 상태 파악 어려움
- 계좌별 예수금과 보유 종목을 일일이 확인해야 함
- 통합 자산 현황을 보려면 엑셀에 수동 입력 필요

#### 1.2 투자 판단 기록 부재
- 매수 이유와 투자 판단이 흩어져 있거나 기록되지 않음
- 시간이 지나면 "왜 이 종목을 샀는지" 기억나지 않음
- 종목을 분석한 내용을 저장할 곳이 없음

#### 1.3 투자 복기 어려움
- 매도 후 "왜 샀었지?"라는 의문에 답할 수 없음
- 과거 투자 판단을 돌아보고 배울 기회 부족
- 실수를 반복하게 됨

#### 1.4 다중 기기 접근 제한
- PC에서 작성한 노트를 모바일에서 볼 수 없음
- 외출 중 투자 판단을 확인하기 어려움
- 기기별로 데이터가 분리되어 관리 불편

### 문제 정의 (압축)
> **개인 투자자는 자신의 주식 종목을 관리하고 싶지만, 자산은 흩어져 있고 투자 판단은 기록되지 않으며, 언제 어디서나 접근하기 어렵다.**

## 2. 사용자 페르소나

### 주 사용자 (Primary Persona)

**이름**: 김투자 (가명)  
**나이**: 30대  
**직업**: 직장인  
**투자 경험**: 2년 (주린이)

#### 투자 프로필
- **투자 성향**: 국내 주식 (AI ETF, 반도체), 미국 주식 (성장주, 배당주)
- **계좌 현황**: 한국투자증권, 유안타증권, 카카오증권 등 3개 증권사
- **투자 금액**: 3,000만원 ~ 5,000만원
- **거래 빈도**: 주 1~2회

#### 행동 패턴
- **사용 시점**: 
  - 평일 장 종료 후 (오후 4시~6시)
  - 주말 자산 정리 시간 (토요일 오전)
  - 매수/매도 직후 (즉시)

- **주요 행동**:
  - 현재 포지션과 계좌별 자산 상태 빠르게 확인
  - 매수/매도 직후 판단과 감정 즉시 기록
  - 이전 투자 판단을 다시 읽으며 복기

#### 니즈 (Needs)
- ✅ 증권사별 자산을 한 화면에서 정리하고 싶음
- ✅ 매수 이유와 투자 판단을 기록하고 싶음
- ✅ 엑셀이나 메모앱보다 구조화된 관리 도구를 원함
- ✅ 종목을 분석한 내용을 쉽게 저장하고 싶음

#### 사용을 방해하는 요인
- ❌ 입력 과정이 번거롭거나 시간이 많이 걸리는 경우
- ❌ 단순 조회용 정보만 제공되어 기록할 이유가 없는 경우
- ❌ HTS/MTS와 차별점이 느껴지지 않는 경우

## 3. 비즈니스 목표 (Goals)

### 3.1 제품 목표
- 여러 증권사 계좌의 예수금과 보유 종목을 논리적으로 통합 관리
- 종목별 투자 판단과 히스토리를 기록할 수 있는 구조 제공
- 언제 어디서나 접근 가능한 클라우드 기반 웹앱 구현
- 안정적인 데이터 동기화 및 자동 백업

### 3.2 단기 목표 (ASAP)
- ✅ 본인이 주 2회 이상 사용
- ✅ 투자 노트 월 10개 이상 작성
- ✅ 2개 이상 기기에서 데이터 동기화 확인
- ✅ 3개 이상 종목에 투자 노트 작성

### 3.3 장기 목표 (6개월)
- 투자 판단 복기를 통한 수익률 개선 체감
- 주변 투자자 3명 이상에게 추천
- 이미지 첨부 기능 활발히 사용 (월 10개 이상)

## 5. 핵심 기능 (MVP Scope)

### 5.1 계좌 관리

#### 기능 설명
- 증권사 이름을 기준으로 계좌 생성
- 계좌별 예수금 수동 관리
- 계좌 추가/수정/삭제

#### 사용자 스토리
**US-001**: 계좌 추가  
- **As a** 투자자
- **I want to** 증권사 계좌를 추가하고
- **So that** 여러 계좌의 예수금을 한 곳에서 관리할 수 있다

**Acceptance Criteria:**
- AC1: 증권사명과 예수금을 입력하여 계좌 추가 가능
- AC2: 계좌 목록에서 전체 예수금 합계 확인 가능
- AC3: 계좌 수정/삭제 가능
- AC4: 계좌 최대 20개까지 추가 가능

#### 제약사항
- 계좌명: 최대 50자
- 계좌 개수: 최대 20개
- 예수금: 0 ~ 999조 원 (정수)
- 통화: KRW만 지원

---

### 5.2 종목 관리

#### 기능 설명
- 관심 종목(WATCHLIST) 등록
- 보유 종목(HOLDING) 관리
- 종목 상태 자동 전환 (매수/매도 시)

#### 사용자 스토리
**US-002**: 관심 종목 등록  
- **As a** 투자자
- **I want to** 관심 있는 종목을 등록하고
- **So that** 리서치 내용을 기록할 수 있다

**Acceptance Criteria:**
- AC1: 종목명, 코드 입력 가능
- AC2: 관심 종목 목록 조회 가능
- AC3: 종목별 리서치 노트 작성 가능

**US-003**: 관심 종목 → 보유 종목 전환  
- **As a** 투자자
- **I want to** 관심 종목을 보유 종목으로 전환하고
- **So that** 매수 정보를 기록할 수 있다

**Acceptance Criteria:**
- AC1: 계좌, 수량, 단가 입력 가능
- AC2: 상태가 WATCHLIST → HOLDING으로 자동 변경
- AC3: 리서치 노트를 매수 이유로 가져오기 가능 (선택)

#### 상태 관리
- **WATCHLIST** (관심): 아직 매수하지 않은 종목
- **HOLDING** (보유): 현재 보유 중인 종목
- **PARTIAL_SOLD** (일부매도): 일부만 매도한 종목
- **SOLD** (전량매도): 전량 매도한 종목

#### 제약사항
- 종목명: 최대 50자
- 종목코드: 6자리 (선택)
- 보유 수량: 0 ~ 999,999 (정수)
- 평균단가: 0 ~ 999,999,999원 (정수)

### 5.3 투자 노트 (핵심 기능)

#### 기능 설명
- 매수/매도/일반 노트 작성
- 구조화된 입력 필드 (매수 이유, 기대 시나리오, 리스크 등)
- 이미지 첨부 (드래그 앤 드롭, 붙여넣기)
- 노트 수정/삭제

#### 사용자 스토리
**US-004**: 투자 노트 작성  
- **As a** 투자자
- **I want to** 매수/매도 판단을 기록하고
- **So that** 나중에 투자 판단을 복기할 수 있다

**Acceptance Criteria:**
- AC1: 노트 타입 선택 가능 (매수/매도/일반)
- AC2: 구조화된 필드에 내용 입력 가능
- AC3: 이미지 첨부 가능 (드래그 앤 드롭, 붙여넣기)
- AC4: 작성한 노트 수정/삭제 가능
- AC5: 종목 상세 화면에서 노트 목록 조회 가능

#### 노트 타입별 필드
**매수 노트 (PURCHASE):**
- 매수 이유 (buyReason)
- 기대 시나리오 (expectedScenario)
- 리스크 (risks)

**매도 노트 (SELL):**
- 매도 이유 (sellReason)
- 매도 후 회고 (sellReview)

**일반 노트 (GENERAL):**
- 현재 생각 (currentThought)

#### 제약사항
- 각 필드: 최대 500자
- 이미지: 노트당 최대 5개
- 이미지 크기: 최대 5MB/파일
- 지원 형식: JPG, PNG, WebP

## 7. 데이터 모델

### User (사용자) - Supabase Auth
```typescript
{
  id: string (UUID, Supabase Auth)
  email: string (이메일)
  name: string | null (이름, 최대 50자)
  createdAt: timestamp
  updatedAt: timestamp
}
```

### Account (계좌)
```typescript
{
  id: string (UUID)
  userId: string (FK → User.id)
  brokerName: string (증권사명, 최대 50자)
  cashBalance: number (예수금, 0 ~ 999,999,999,999)
  memo: string | null (메모, 최대 200자)
  createdAt: timestamp
  updatedAt: timestamp
}
```

### Stock (종목)
```typescript
{
  id: string (UUID)
  accountId: string | null (FK → Account.id, WATCHLIST인 경우 null)
  symbol: string | null (종목코드, 6자리)
  name: string (종목명, 최대 50자)
  quantity: number (수량, 0 ~ 999,999)
  avgPrice: number (평균단가, 0 ~ 999,999,999)
  status: enum ('WATCHLIST' | 'HOLDING' | 'PARTIAL_SOLD' | 'SOLD')
  sector: string | null (섹터)
  industry: string | null (산업)
  createdAt: timestamp
  updatedAt: timestamp
}
```

### StockMemo (투자 노트)
```typescript
{
  id: string (UUID)
  stockId: string (FK → Stock.id)
  type: enum ('PURCHASE' | 'SELL' | 'GENERAL')
  buyReason: string | null (최대 500자)
  expectedScenario: string | null (최대 500자)
  risks: string | null (최대 500자)
  currentThought: string | null (최대 500자)
  sellReview: string | null (최대 500자)
  createdAt: timestamp
  updatedAt: timestamp
}
```

### Attachment (첨부 파일) - Supabase Storage
```typescript
{
  id: string (UUID)
  memoId: string (FK → StockMemo.id)
  type: enum ('IMAGE')
  fileName: string (원본 파일명, 최대 255자)
  fileSize: number (바이트, 최대 10MB)
  mimeType: string (예: 'image/jpeg')
  storageUrl: string (Supabase Storage Public URL)
  createdAt: timestamp
}
```

**관계:**
- User 1 : N Account
- Account 1 : N Stock
- Stock 1 : N StockMemo
- StockMemo 1 : N Attachment

## 11. 성공 기준

### 정량적 기준 (ASAP)
- ✅ 최소 3개 이상의 종목에 대해 투자 노트 작성
- ✅ 관심 종목 최소 2개 이상 등록 및 리서치 자료 첨부
- ✅ 노트에 이미지 첨부 최소 5회 이상
- ✅ 1주일에 최소 2회 이상 앱 접속 (2주 연속)
- ✅ 작성한 노트를 최소 1회 이상 다시 조회
- ✅ 2개 이상 기기에서 데이터 동기화 확인
- ✅ 로그인/로그아웃 정상 동작

### 정성적 기준
- "이번 주말에 자산 정리하려면 이 앱을 켜야지"라는 생각이 듦
- 매도 후 "왜 샀었지?"라는 의문이 들 때 앱에서 답을 찾을 수 있음
- 종목 분석 내용을 자연스럽게 앱에 저장하게 됨
- HTS보다 투자 판단 관리에 유용하다고 느낌
- 외출 중에도 모바일에서 투자 노트를 확인할 수 있음
